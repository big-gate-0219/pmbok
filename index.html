                    １０の知識ベース
                    統合マネジメント
                    1. プロジェクト憲章作成
                    2. プロジェクトマネジメント計画書作成
                    3. プロジェクト作業の指揮・マネジメント
                    4. プロジェクト作業の監視・コントロール
                    5. 統合変更管理
                    6. プロジェクトやフェーズの集結
                    スコープ・マネジメント
                    プロジェクトの達成に必要な作業と成果物を確定させるためにプロセスを定義。
                    1. スコープ・マネジメント計画
                    2. 要求事項収集
                    3. スコープ定義
                    4. WBS作成
                    5. スコープ妥当性確認
                    6. スコープ・コントロール
                    タイム・マネジメント
                    プロジェクトに達成すべきスケジュール内で完了させるために必要なプロセスを定義。
                    1. スケジュール・マネジメント計画
                    2. アクティビティ定義
                    3. アクティビティ順序設定
                    4. アクティビティ資源見積り
                    5. アクティビティ所要期間見積り
                    6. スケジュール作成
                    7. スケジュール・コントロール
                    コスト・マネジメント
                    プロジェクトを予算内で達成するために必要なプロセスを定義。
                    1. コスト・マネジメント計画
                    2. コスト見積り
                    3. 予算設定
                    4. コスト・コントロール
                    品質・マネジメント
                    プロジェクトが達成すべき品質を持ち、管理するために必要なプロセスを定義。
                    1. 品質マネジメント計画
                    2. 品質保証
                    3. 品質コントロール
                    人的資源・マネジメント
                    プロジェクト・チームを編成・育成・管理し、目標を達成するために必要なプロセスを定義。
                    1. 人的資源マネジメント計画作成
                    2. プロジェクト・チーム編成
                    3. プロジェクト・チーム育成
                    4. プロジェクト・チーム・マネジメント
                    コミュニケーション・マネジメント
                    情報伝達や円滑なコミュニケーションに必要なプロセスを定義。
                    1. コミュニケーション・マネジメント計画
                    2. コミュニケーション・マネジメント
                    3. コミュニケーション・コントロール
                    リスク・マネジメント
                    リスクの特定・分析・対応・監視に必要なプロセスを定義。
                    1. リスク・マネジメント計画
                    2. リスク特定
                    3. 定性的リスク分析
                    4. 定量的リスク分析
                    5. リスク対応計画
                    6. リスク・コントロール
                    調達・マネジメント
                    プロジェクト実施に必要なプロダクトやサービスについて、調達の計画・実行・管理・集結に必要なプロセスを定義。
                    1. 調達マネジメント計画
                    2. 調達実行
                    3. 調達コントロール
                    4. 調達集結
                    ステークホルダー・マネジメント
                    プロジェクト実施に必要なステークホルダーの特定とその要求の把握、利害の調整に必要なプロセスを定義。
                    1. ステークホルダー・マネジメント計画
                    2. ステークホルダー・エンゲージメント・マネジメント
                    3. ステークホルダー・エンゲージメント・コントロール

                    ５つのプロセス群

                    ５つのプロセス群
                      定義 内容
                    １ 立ち上げ プロジェクトまたはプロジェクトの新しいフェーズを明確に定め、それらを開始する許可を得るプロセス
                    ２ 計画 作業全体のスコープを確定し目標の定義と洗練を行い、目標を達成するのに必要な一連の行動の流れを規定するプロセス
                    ３ 実行 プロジェクト目標を達成する上でプロジェクトマネジメント計画書において規定された作業を実行するプロセス
                    ４ 監視・コントロール プロジェクトの進捗やパフォーマンスの追跡、レビュー、統制、計画の変更が必要な分野の特定、およびそれらの変更を開始するプロセス
                    ５ 集結 プロジェクトやフェーズを公式に終了するための、全プロジェクトマネジメント・プロセス群内の全アクティビティを集結するためのプロセス
