メモ

フェーズの目的

  1. 解決すべきビジネス上の問題の明確化
  2. 実行可能なテクニカルソリューションの識別
  3. アプローチの計画
  4. 作業環境とチームの立ち上げ
  5. 選ばれた戦略の実施に対する投資を続行しても良いという利害関係者の同意の取り付け

典型的なプロジェクト立ち上げ（４週間）

  1. 初期の要求構想
  2. 初期のアーキテクチャの構想
  3. チームの編成
  4. 初期のリリース計画

方向付けフェーズのゴール

  1. 初期チームの編成
  2. プロジェクトのビジョンを特定
  3. 利害関係者のビジョンの合意を取り付ける
  4. 企業の方針に準ずる
  5. 初期の技術戦略、初期の要求、初期のリリース計画を策定する
  6. 作業環境を構築する
  7. 予算確保
  8. リスクの特定

方向付けフェーズのアプローチ

  1. 調整
    1. チームの立ち上げ
    2. 構想選定セッションのための利害関係者収集
  2. 協働
    1. チームの確立
    2. 要求の構想
    3. アーキテクチャーの構想
    4. 実現可能性の検討
    5. エンタープライズ戦略との調整
    6. リリース計画（初期）
    7. 共有ビジョンの策定
    8. 環境構築
  3. 完成
    1. マイルストーンレビュー（簡易）
    2. 利害関係者とのビジョンの合意

作業内容もしくは作業成果物

  1. 開発構想書, (OPT)プロジェクト憲章, (OPT)ビジネスケース
  2. 合意形成
  3. 初期スコープの識別
    1. 初期要求
    2. 要求モデリング
    3. モデリング戦略
    4. ワークアイテム管理戦略
    5. 非機能要求の戦略
  4. 初期の技術戦略
    1. アーキテクチャレベル
    2. アーキテクチャモデリング
    3. モデリング戦略
  5. 初期のリリース計画
  6. 作業環境の構築
